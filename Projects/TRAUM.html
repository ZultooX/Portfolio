TRAUM
Being the first project made outside Unity and in a custom engine it was extra fun to work on. The engine used is TGE, an engine made and maintainded by the educators from The Game Assembly. The engine is striped down to the bare minimum. This was one of the longest projects, about 12 weeks on half time.

Scripted Animation

The monster we had had to follow the player in a way that makes sense and should not collide with the ground. Doing this only with code would be alot of code in a single script.

Unity has a way to animate an objects Transform and outputs it to a clean .animation text file. This file has all info about all keyframes to know what the transform should be at that point.

All that had to be done is extracting all keyframes and getting the transform for it. With the list we can trigger the animation sequence and the object transforms along the animation
After the project finished I do think that making a simple json from the .animation file would be more optimized for file size, since the .animation file contains much more information we did not need

Sprite Sheet Extractor

Manually defining all edges of all sprites in a spritesheet can take very long. If you have different sizes as well you cant simply repeat the same shape. In Unity there's a sprite editor where they solve this issue.

All these rectangles goes to a .meta file next to the image and are all written in plain text. Taking all these and importing them to our game we now have a sequence of sprites that can be animated

Sprite Animator

Easiest way to animate an image is to have multipe images and switdch them fast, like how GIFs or Videos are made. It works for files but in games its not very viable, that's why sprite sheet animations are so great, they solve this.

Using Min and Max Coordinates on the sprite sheet we can tell the Renderer to only render what is inside these coordinates. Then switch every X Seconds for an animation.

Closing Thoughts
